=== Front18 Security Integration Changelog ===

= 1.1.2 =
* Feature: novo endpoint público GET /wp-json/front18/v1/ping — health-check para o SaaS confirmar que o plugin está instalado e ativo.
* O /ping retorna: version, status (ok / degraded / disabled), enabled, synced e last_sync.
* status 'ok' = plugin ativo e API Key configurada.
* status 'degraded' = plugin ativo mas sem API Key (aguardando configuração inicial).
* status 'disabled' = plugin instalado mas desativado no painel WordPress.
* UX: seção de Shortcodes no painel admin reescrita com linguagem mais didática e badges por perfil de uso.
* UX: Painel de Desenvolvedor renomeado para "Configurações Avançadas" com avisos sobre uso em produção.

= 1.1.1 =
* Hotfix: substitui placeholder %i do $wpdb->prepare() por interpolação direta — corrige Fatal Error em WordPress < 6.2.
* O placeholder %i para nomes de tabela foi introduzido apenas no WordPress 6.2 (maio 2023). Sites em versões anteriores recebiam um erro fatal ao ativar o plugin após a atualização 1.1.0.

= 1.1.0 =
* Segurança: hash_equals() na verificação da API Key — previne timing attacks.
* Segurança: $wpdb->prepare() em todas as queries diretas na classe API.
* Segurança: uninstall.php atualizado para limpar todas as opções das versões 1.0.x e 1.1.0.
* Segurança: $wpdb->prepare() na query de limpeza de postmeta no uninstall.
* Performance: get_option() centralizado em propriedades com cache na classe Frontend (0 chamadas duplicadas).
* Performance: update_option() removido da _calculate_scope() — migração de dados legados movida para hook de ativação/atualização.
* Performance: Ghost Tracker agora usa cache via transient de 12h (invalidado automaticamente no webhook do SaaS).
* Ghost Tracker v3: detecta imagens em ACF, WooCommerce (product galleries), Divi e theme_mods além do Elementor.
* Ghost Tracker v3: queries em lotes de 50 registros (batch) para controle de memória em sites grandes.
* API REST: filtros ricos de mídia — tipo MIME, dimensões mínimas/máximas, campo de ordenação, direção.
* API REST: metadados ricos no retorno — filesize, width, height, alt text, mime_type, is_ghost.
* API REST: novo endpoint GET /wp-json/front18/v1/media/types — lista tipos MIME disponíveis no site.
* API REST: args declarativos em todos os endpoints (auto-documentação e validação antecipada de parâmetros).
* Admin: card "Proteção Ativa Agora" — resumo visual de modo, nível, quantidade de mídias e escopo.
* Admin: seção "Shortcodes Disponíveis" — documenta [front18], [front18_lock] e data-front18="locked".
* Admin: meta box mostra status atual da proteção da página considerando regras globais e de CPT.
* Cache buster do SDK agora gerado via PHP com gmdate('Ymd') — consistente em ambientes multi-servidor.
* Testado até WordPress 6.7.
